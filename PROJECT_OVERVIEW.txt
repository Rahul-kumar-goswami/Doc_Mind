=========================================
      PROJECT OVERVIEW: DocuMind Enterprise
=========================================

DocuMind Enterprise is a Retrieval-Augmented Generation (RAG) based document assistant 
built using Django and LangChain. It allows users to upload documents (PDF/DOCX), 
which are then indexed for fast similarity search, enabling the assistant 
to answer questions based on the document content.

STEP-BY-STEP COMPONENTS:

1. CORE TECHNOLOGIES & FRAMEWORKS
---------------------------------
- Backend: Django 6.0 (Python)
- AI Orchestration: LangChain
- Vector Database: FAISS (Facebook AI Similarity Search)
- LLM: Llama 3.1 8B (via Groq API)
- Embeddings: HuggingFace (sentence-transformers/all-MiniLM-L6-v2)
- Database: SQLite3

2. PROJECT STRUCTURE
--------------------
- /ai_project: Main Django configuration (settings, URLs, WSGI).
- /assistant: Primary application containing:
    - models.py: Custom User, Chat History, and User Memory models.
    - views.py: Request handling, authentication logic, and chat logic.
    - rag_engine.py: The AI brain of the project.
- /templates: HTML frontend divided into registration and assistant views.
- /static: CSS and JS files for styling and client-side logic.
- /media: Storage for uploaded documents.
- /faiss_index: Local storage for the vector database.
- /model_cache: Local cache for downloaded embedding models.

3. THE RAG ENGINE (AI LOGIC)
----------------------------
- Document Ingestion:
    1. Loads files using PyPDFLoader or Docx2txtLoader.
    2. Cleans text using a custom 'de-space' heuristic (fixes PDF spacing issues).
    3. Splits text into chunks (1000 characters) using RecursiveCharacterTextSplitter.
    4. Generates embeddings and saves them to the FAISS index.
- Retrieval & Answer:
    1. Receives user query.
    2. Searches the FAISS index for the top 3 most relevant document chunks.
    3. Constructs a prompt (DocuMind Enterprise Personality) including context and chat history.
    4. Sends the prompt to Groq (Llama 3.1) to generate a friendly response.

4. AUTHENTICATION & SECURITY
----------------------------
- Custom User System: Uses a CustomUser model instead of default Django User.
- Password Security: Uses Bcrypt for secure password hashing.
- Signup Flow: Email -> Password -> OTP Verification (sent via SMTP).
- Login Security: Includes an alphanumeric CAPTCHA to prevent automated logins.
- Password Reset: OTP-based reset flow with CAPTCHA validation.

5. USER EXPERIENCE & FEATURES
-----------------------------
- Chat History: Saves user and assistant messages to provide continuity.
- User Memory: Remembers facts about the user (e.g., their name).
- Responsive UI: Styled with custom CSS for a modern, friendly look.
- Interactive Feedback: Emojis and warm greetings integrated into AI responses.

6. SETUP & CONFIGURATION
------------------------
- Dependencies: Listed in 'requirements.txt'.
- Environment Variables: '.env' file stores the GROQ_API_KEY.
- Static Files: Managed via 'whitenoise' for production compatibility.
